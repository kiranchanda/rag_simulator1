Learn RAG step by step

RAG, or Retrieval-Augmented Generation, helps an AI answer questions using your own documents instead of relying only on what the model already knows.

What this simulator shows

Paste knowledge, split it into chunks, create fake embeddings, store them in a simulated vector database, retrieve the best chunks, and generate a grounded answer.

Knowledge Base

Add documents, emails, PDFs text, chats, help articles, policies, or product data. This becomes the source knowledge for RAG.

Important: RAG does not usually retrain the LLM. It retrieves relevant context and sends that context to the LLM at question time.

Step 1 — Ingest source knowledge

RAG starts by collecting trusted information from your business systems and documents.

1
Inputs

Documents, emails, PDFs, chats

These are your raw sources of truth.

Preparation

Clean and normalize content

Remove noise, extract text, preserve metadata, and keep source references.

Step 2 — Create chunks

Large documents are split into smaller pieces so the system can retrieve the most relevant sections later.

2

Step 3 — Create vector embeddings

Each chunk is converted into a numeric vector. In a real system, an embedding model creates these vectors.

3

Step 4 — Store in a vector database

The vector database stores vectors, chunk text, and metadata so the system can search by meaning.

4
Vector DB Status

0 records stored

Each record contains chunk text, vector values, and metadata.

Example Metadata

source, chunk_id, text, vector

Metadata enables filtering, permissions, traceability, and citations.

Why this matters

The LLM does not need to memorize your documents

Your knowledge stays outside the model. The app retrieves only the most relevant chunks when a user asks a question.

Step 5 — Retrieve relevant chunks

The user question is embedded, then compared against stored vectors to find the most relevant chunks.

5

Step 6 — Generate a grounded answer

The retrieved chunks are sent to the LLM as context. The LLM then answers using that context.

6
Production best practice: Show citations, check freshness, enforce permissions, evaluate answer quality, and escalate uncertain answers to a human.
